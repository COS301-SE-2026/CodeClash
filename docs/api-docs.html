Overview

Base URL and general conventions for the CodeClash REST API.

Base URL

http://localhost:3000/api

Content Type

application/json

Auth Header

Authorization: Bearer <token>

Response Codes

200 OK 201 Created 400 Bad Request 401 Unauthorized 404 Not Found 409 Conflict 500 Server Error

Auth Flow

CodeClash uses JWT bearer tokens. Register or login to receive a token, then include it in the Authorization header for protected endpoints.

POST /auth/register Register a new user

Creates a new user account and returns a JWT token.

Request body
FieldType
usernamestringrequired
emailstringrequired
passwordstringrequired
Response 201
json
{
  "message": "User registered successfully",
  "user": {
    "user_id": "uuid",
    "username": "player1",
    "email": "player1@email.com"
  }
}
POST /auth/login Login and get token

Authenticates a user and returns a JWT token valid for 24 hours.

Request body
FieldType
emailstringrequired
passwordstringrequired
Response 200
json
{
  "message": "Login successful",
  "token": "eyJhbGci..."
}

ELO Ratings

Manage and query ELO ratings. Ratings start at 600 and are updated automatically after each ranked match using the standard Elo formula with K=32.

GET /elo/{user_id} Get user's current ELO Auth

Returns the current ELO rating for a user.

Path parameters
ParamType
user_idUUIDrequired
Response 200
json
{
  "elo_id": "uuid",
  "user_id": "uuid",
  "username": "player1",
  "rating": 616,
  "updated_at": "2026-05-01T12:00:00Z"
}
GET /elo/{user_id}/history Get ELO history Auth

Returns full ELO change history for a user, ordered by most recent first.

Path parameters
ParamType
user_idUUIDrequired
Response 200
json
[
  {
    "history_id": "uuid",
    "user_id": "uuid",
    "match_id": "uuid",
    "old_rating": 600,
    "new_rating": 616,
    "change": 16,
    "changed_at": "2026-05-01T12:00:00Z"
  }
]
POST /elo/update Update ELO after ranked match Auth

Updates ELO ratings for both players after a ranked match. Uses the standard Elo formula (K=32). Only applies to ranked matches — returns 400 for casual matches. Wrapped in a database transaction.

Request body
FieldType
match_idUUIDrequired
winner_idUUIDrequired
loser_idUUIDrequired
Response 200
json
{
  "message": "Elo ratings updated",
  "winner": {
    "user_id": "uuid",
    "old_rating": 600,
    "new_rating": 616,
    "elo_gained": 16
  },
  "loser": {
    "user_id": "uuid",
    "old_rating": 600,
    "new_rating": 584,
    "elo_lost": 16
  }
}
GET /elo/leaderboard Get top 10 players

Returns the top 10 players ranked by ELO rating, with their rank position included.

Response 200
json
[
  {
    "user_id": "uuid",
    "username": "player1",
    "rating": 820,
    "updated_at": "2026-05-01T12:00:00Z",
    "rank": 1
  }
]

Matches

Matches are created by the matchmaking system in the frontend. Each match links exactly two players, belongs to a game mode, and tracks status from waiting through to completed.

GET /matches List matches Auth

Returns a paginated list of matches for the authenticated user. Optionally filter by status or mode.

Query parameters
ParamType
statusstringoptional
modestringoptional
limitintegeroptional
offsetintegeroptional
Response 200
json
[
  {
    "match_id": "uuid",
    "player1_id": "uuid",
    "player2_id": "uuid",
    "mode": "ranked",
    "status": "completed",
    "match_start": "2026-05-01T12:00:00Z"
  }
]
GET /matches/{match_id} Get match details Auth

Returns full details for a single match including players, problems, status, and log.

Response 200
json
{
  "match_id": "uuid",
  "player1_id": "uuid",
  "player2_id": "uuid",
  "match_problems_id": "uuid",
  "mode": "ranked",
  "status": "in_progress",
  "queue_start": "2026-05-01T11:58:00Z",
  "match_start": "2026-05-01T12:00:00Z"
}
POST /matches Create a match Auth

Creates a new match between two players. Called internally by the matchmaking system once two players are paired.

Request body
FieldType
player1_idUUIDrequired
player2_idUUIDrequired
modestringrequired
match_problems_idUUIDoptional
Response 201
json
{
  "message": "Match created",
  "match_id": "uuid",
  "status": "waiting"
}
PATCH /matches/{match_id}/status Update match status Auth

Updates the status of a match. Valid transitions: waiting → in_progress → completed | abandoned.

Request body
FieldType
statusstringrequired

status: waiting | in_progress | completed | abandoned

Response 200
json
{
  "message": "Match status updated",
  "match_id": "uuid",
  "status": "in_progress"
}
GET /matches/{match_id}/log Get match log Auth

Returns the match log including winner, loser, and ELO changes. ELO values are null for casual matches.

Response 200
json
{
  "log_id": "uuid",
  "match_id": "uuid",
  "winner_id": "uuid",
  "loser_id": "uuid",
  "elo_gained": 16,
  "elo_lost": 16
}

GET /problems List problems

Returns all problems. Filter by type or difficulty.

Query parameters
ParamType

difficultystringoptional

type: math | programming
difficulty: Easy | Medium | Difficult

Response 200
json

    "id": 1,
    "type": "math",

    "title": "Basic Algebra",
    "description": "Solve for x",
    "time_limit": "00:02:00"
  }
]
GET /problems/{id} Get problem details

Returns full details for a single problem including type-specific fields.

Response 200
json
{
  "id": 1,
  "type": "programming",

  "description": "Return nth Fibonacci",

Request body
FieldType

titlestringrequired

Response 200

DELETE /problems/{id} Delete a problem Admin

json
{ "message": "Problem deleted" }

GET /problems/{problem_id}/testcases List test cases Auth

Returns all test cases for a given programming problem.

Response 200
json
[
  {
    "testcase_id": "uuid",
    "problem_id": 1,
    "input": "10",
    "expected_output": "55"
  }
]
POST /problems/{problem_id}/testcases Add a test case Admin

Adds a new test case to a programming problem.

Request body
FieldType
inputstringrequired
expected_outputstringrequired
Response 201
json
{ "message": "Test case added" }
PUT /problems/{problem_id}/testcases/{testcase_id} Update a test case Admin

Response 200

Submissions

Code and math answer submissions made by players during a match. Submissions are evaluated by the Judge0 execution engine.

POST /submissions Submit a solution Auth

Submits a user's code or math answer for evaluation. Triggers Judge0 execution for programming submissions.

Request body
FieldType
match_idUUIDrequired
problem_idintegerrequired
codestringrequired
languagestringrequired
Response 201
json
{
  "submission_id": "uuid",
  "status": "Pending",
  "submitted_at": "2026-05-01T12:05:00Z"
}
GET /submissions/{submission_id} Get submission status Auth

Returns the current status of a submission. Poll this after submitting to check if execution has completed.

Response 200
json
{
  "submission_id": "uuid",
  "status": "Correct",
  "submitted_at": "2026-05-01T12:05:00Z"
}

status: Pending | Correct | Incorrect | Error

GET /users/{user_id}/submissions List user submissions Auth

Returns all submissions made by a user, ordered by most recent first.

Response 200
json
[
  {
    "submission_id": "uuid",
    "match_id": "uuid",
    "status": "Correct",
    "submitted_at": "2026-05-01T12:05:00Z"
  }

Execution Results

Detailed results from the Judge0 code execution engine, including pass/fail counts, execution time, and memory usage.

GET /submissions/{submission_id}/result Get execution result Auth

Returns the full execution result for a submission once Judge0 has completed evaluation.

Response 200
json
{
  "result_id": "uuid",
  "submission_id": "uuid",
  "passed_cases": 5,
  "total_cases": 5,
  "execution_time": 142,
  "memory_used": 20480
}

Error Codes

All errors return a consistent JSON body with a machine-readable message field.

Error envelope
{
  "message": "Elo is only updated for ranked matches"
}

HTTPWhen
400Missing or invalid request body fields
401Missing or invalid bearer token
403Token valid but insufficient permissions
404Resource not found
409Conflict — e.g. duplicate username or email
500Internal server error — check server logs
